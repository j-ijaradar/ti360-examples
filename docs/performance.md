# Performance — numbers, not adjectives

Every performance claim on this page links to a committed, regenerable
measurement. If a number has no benchmark behind it, it does not belong
here (plan principle 10).

## Out-of-core scaling

The streaming kernels (`core/src/stream.rs`) hold per-bucket aggregation
state instead of the input, so memory scales with the output grid:

- **10× the input rows costs 1.29× the streaming peak memory** (vs 2.05×
  for the in-memory path on the same machine).
- The streamed result is **cell-for-cell identical** to the in-memory
  kernel (asserted in Rust and Python tests).

Source: [`benchmarks/perf/baseline.md`](https://github.com/j-ijaradar/trafficinsight360-dev/blob/master/benchmarks/perf/baseline.md),
regenerated with `python benchmarks/perf/run.py --write-baseline`. CI
re-runs the measurement and fails if the scaling ratio reaches 5×
(ratios, unlike absolute MB/s, are hardware-independent).

## Kernel benchmarks (criterion)

`cargo bench -p trafficinsight360-core` measures the hot paths on the
bundled sample (2 592 rows, 3 sensors, 3 measures); reference numbers
from the baseline machine:

| benchmark | time |
|---|---|
| `resample_mean` to 900 s grid | ~0.38 ms |
| full `advise` (fingerprint + KB + zoo matching) | ~1.5 ms |

Machine-specific — rerun locally; criterion stores its own history under
`target/criterion` for regression comparison.

## Advisor quality (measured, not assumed)

The evaluation harness compares library defaults, advisor-recommended
pipelines and an oracle sweep on three benchmark datasets with
constructed ground truth:

- **+32.8 % mean improvement over defaults**, zero advised-below-default
  rows, mean regret vs oracle 8.4 %.

Source: [`benchmarks/advisor/report_card.md`](https://github.com/j-ijaradar/trafficinsight360-dev/blob/master/benchmarks/advisor/report_card.md)
— regenerated in CI on every knowledge-base change; regressions fail CI.

## Cross-language overhead

Python, R and the CLI call the same core; the bindings' overhead is one
JSON serialization of the result document. Byte-identical outputs are
enforced in CI for profile, validate, fingerprint, advise, health, zoo
inference and pipeline manifests — parity is a correctness guarantee
first, but it also means benchmark numbers measured in one language hold
for the others.
